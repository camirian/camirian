# AGENTS.md

This repository follows the `system-blueprint` operating standard.

## Rules

- Treat `README.md` as the user entrypoint and keep it accurate.
- Keep docs aligned with the actual repo behavior and file layout.
- Prefer the smallest change that solves the stated problem.
- Preserve unrelated user work; do not revert changes you did not make.
- Verify with the lightest checks that cover the change.
- Update docs when behavior, entrypoints, or operating assumptions change.

## Agent workflow

1. Inspect the repo before changing it.
2. Make the minimal safe edit.
3. Verify the change.
4. Leave a clear handoff in the repo docs when the workflow changes.
