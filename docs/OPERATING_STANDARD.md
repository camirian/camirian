# Operating Standard

## Purpose

This repository is managed as part of the broader `/home/caaren/dev` workspace. The expectation is professional, documented, and reproducible work.

## Defaults

- Keep the terminal baseline boring and stable.
- Use `zellij` only when persistent sessions or multi-pane work actually help.
- Prefer reversible changes over speculative refactors.
- Keep strategy notes separate from execution artifacts.

## Documentation rules

- `README.md` explains what the repo is and how to start.
- `AGENTS.md` explains how Codex and Claude Code should behave here.
- Add a short runbook or quickstart when the repo can actually be run.
- Keep examples current with the real files and commands in the repo.

## Change process

1. Inspect the current state.
2. Make the smallest change that satisfies the need.
3. Verify the change with the narrowest useful check.
4. Update docs if the behavior or workflow changed.
5. Leave the repo easier to resume than you found it.
